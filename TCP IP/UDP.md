# [Study] UDP 프로토콜과 다자간 통신 (Broadcasting)

## 39. UDP (User Datagram Protocol) 개요
* **TCP와 UDP의 공통점**:
    1. **포트(Port) 번호**를 이용하여 통신의 종착점(응용 프로그램 주소)을 지정함.
    2. 헤더에 포함된 체크섬(Checksum)을 통해 **데이터 오류를 체크**함.

### 39.1 UDP의 4가지 핵심 특징
1. **비연결성**: 연결 설정(3-Way Handshake)을 하지 않으므로 TCP의 `connect()` 함수 호출이 불필요함.
2. **신뢰성 미보장**: 프로토콜 수준에서 신뢰성 있는 데이터 전송을 보장하지 않음 (패킷 유실 시 재전송 안 함). 
   * *해결책*: 필요하다면 응용 프로그램 수준(Application Layer)에서 확인 응답(ACK) 등의 신뢰성 전송 기능을 직접 구현해야 함.
3. **다자간 통신 용이**: 간단한 소켓 함수 호출 절차만 따르면 1:N, N:M 등 다자간 통신을 매우 쉽게 구현할 수 있음.
4. **데이터 경계 유지**: TCP(스트림 방식)와 달리 패킷(데이터그램) 자체가 독립적인 경계를 가지므로, 응용 프로그램이 데이터 경계 구분을 위한 작업을 별도로 할 필요가 없음.

## 40. TCP vs UDP 비교 총정리 

| 구분 | TCP (Transmission Control Protocol) | UDP (User Datagram Protocol) |
| :--- | :--- | :--- |
| **연결 방식** | **연결형 프로토콜** (연결 설정 후 통신 가능) | **비연결형 프로토콜** (연결 설정 없이 즉시 통신 가능) |
| **신뢰성** | **신뢰성 보장** (필요시 데이터 자동 재전송) | **신뢰성 없음** (데이터 재전송하지 않음) |
| **통신 형태** | 1:1 통신 전용 (Unicast) | 1:1 통신 및 1:多, 多:多 통신 모두 가능 |
| **데이터 서비스** | **데이터 경계 구분 안 함** (바이트 스트림 서비스) | **데이터 경계를 명확히 구분함** (데이터그램 서비스) |

## 41. UDP 데이터 전송 함수
* **`sendto()`**: 목적지 주소(IP/Port)를 지정하여 응용 프로그램의 데이터를 운영체제의 '송신 버퍼'에 복사함으로써 데이터를 전송함.
* **`recvfrom()`**: 운영체제의 '수신 버퍼'에 도착한 데이터를 응용 프로그램 버퍼에 복사함 (송신자의 주소 정보도 함께 받아옴).

## 42. UDP 서버/클라이언트의 IPv6 마이그레이션
> *기존 IPv4 기반 UDP 코드를 IPv6로 전환하기 위한 필수 변경 사항*

1. **새로운 헤더 파일 추가**: 윈도우 환경에 한해 `ws2tcpip.h` 헤더를 추가로 포함해야 함.
2. **소켓 생성 체계 변경**: `socket()` 생성 시 `AF_INET` 대신 **`AF_INET6`**를 사용함.
3. **소켓 주소 구조체 변경**: 16바이트 크기의 `sockaddr_in` 대신, 128비트 주소를 수용할 수 있는 28바이트 크기의 **`sockaddr_in6`** 구조체를 사용함.

## 43. 네트워크 캐스팅 (통신 개체 간 상호 작용 방식) 
| 캐스팅 종류 | 통신 대상 | 개념 요약 |
| :--- | :--- | :--- |
| **유니캐스팅 (Unicasting)** | **1:1** | 특정 목적지 하나를 지정하여 통신하는 방식. |
| **브로드캐스팅 (Broadcasting)** | **1:전체** | 상대방 전체(동일 네트워크 내)에게 통신하는 방식. |
| **멀티캐스팅 (Multicasting)** | **1:특정 그룹** | 내가 지정한 특정 다수(Multicast Group)에게만 통신하는 방식. |
| **애니캐스팅 (Anycasting)** | **1:가장 가까운 1** | 점 구조 (가장 가까운 노드 하나로 라우팅됨). |

## 44. 브로드캐스팅 (Broadcasting) 심화
* **개념**: 송신자가 보낸 데이터 하나를 다수의 수신자가 받는 방식. 송신자 관점에서는 데이터 복사본을 직접 여러 개 만들어 보내는 것이 아니므로(라우터/스위치가 복사함) 네트워크 대역폭 측면에서 상당히 효율적인 기술임.

### 44.1 브로드캐스트 데이터 전송 절차
1. **브로드캐스팅 활성화**: 소켓의 옵션을 변경하여 브로드캐스트 전송을 허용해야 함. *(아키텍트 노트: C언어에서는 `setsockopt()` 함수를 호출하여 `SO_BROADCAST` 옵션을 활성화합니다.)*
2. **목적지 설정 및 전송**: 목적지 IP를 브로드캐스트 주소로 설정하여 `sendto()` 데이터 전송.

### 44.2 브로드캐스트 주소의 종류
1. **네트워크 브로드캐스트 (Local Broadcast)**: 현재 내가 속한 동네(로컬 물리 네트워크)의 모든 호스트에게 묻지마 전송. (주소 예: `255.255.255.255`)
2. **서브넷 브로드캐스트 (Subnet-directed)**: 특정 원격 네트워크(다른 동네) 단위에 있는 '모든' 호스트에게 전송.
3. **지역 브로드캐스트 (All-subnets-directed)**: 원격 네트워크를 더 잘게 쪼갠 '특정 서브넷(구역)' 내의 모든 호스트에게 전송.
