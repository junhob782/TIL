## 1. 네트워크 통신 기초
* **호스트 (Host)**: 네트워크의 가장자리에 위치하여, 최종 사용자의 응용 프로그램(Application)을 수행하는 주체 (예: PC, 스마트폰, 서버).
* **라우터 (Router)**: 호스트에서 생성된 데이터를 여러 네트워크를 거쳐 전송하며, 서로 다른 네트워크에 속한 호스트 간에 데이터를 교환할 수 있게 경로를 찾아주는(Routing) 장비.
* **통신 프로토콜 (Protocol)**: 호스트-라우터, 라우터-라우터, 호스트-호스트 간에 데이터를 안전하고 정확하게 주고받기 위해 사전에 정해진 **절차와 규칙**.

---

## 2. TCP/IP 프로토콜 스택 
* **정의**: 인터넷의 핵심 프로토콜인 **TCP**와 **IP**를 비롯한 각종 통신 규약의 모음.
* **특징**: 운영체제(OS)의 커널(Kernel) 내부 일부로 구현되며, 응용 프로그램(User Application)은 운영체제가 제공하는 소켓(Socket) API를 통해 TCP/IP 프로토콜 서비스를 이용해 통신함.

### 5.1 인터넷 패킷 전송 형태 (TCP/IP 4계층)
1.  **응용 계층 (Application) & 전송 계층 (Transport)**: 하부 계층이 제공하는 '가상적인 연결(Virtual Connection)'을 사용하여 논리적으로 동작함. (TCP/UDP 통신 처리)
2.  **인터넷 계층 (Internet)**: **IP 주소**와 라우팅(Routing) 기능을 이용하여 패킷의 최적 전송 경로를 결정함.
3.  **네트워크 접근 계층 (Network Access)**: MAC 주소(물리 주소)를 사용하여 실제 케이블이나 무선망을 통해 패킷을 전송함.

---

## 3. TCP vs UDP 전송 계층 프로토콜 비교 

| 비교 항목 | TCP (Transmission Control Protocol) | UDP (User Datagram Protocol) |
| :--- | :--- | :--- |
| **연결 방식** | **연결형 프로토콜** (3-Way Handshake 등 연결 설정 후 통신) | **비연결형 프로토콜** (연결 설정 과정 없이 즉시 송신) |
| **신뢰성** | **신뢰성 높음** (오류 발생 시 패킷을 재전송함) | **신뢰성 낮음** (데이터 유실 시 재전송하지 않음) |
| **통신 형태** | 1:1 통신 (Unicast) | 1:1 통신, 1:N 통신 (Broadcast, Multicast 지원) |
| **데이터 처리** | **데이터 경계 구분 안 함** (바이트 스트림 서비스 - Byte Stream) | **데이터 경계 구분함** (데이터그램 서비스 - Datagram) |

---

## 4. 패킷 (Packet) 구조 
* **정의**: 네트워크를 통해 전송되는 데이터의 기본 형식(조각).
* **구성 요소**:
    * **제어 정보**: 각 프로토콜에서 정의한 메타데이터 (예: 출발지/목적지 IP 주소, 포트 번호, 오류 체크 코드 등).
    * **데이터 (Payload)**: 실제 전송하고자 하는 응용 프로그램의 정보.
* **위치에 따른 분류**:
    * **헤더 (Header)**: 데이터의 **앞쪽**에 붙는 제어 정보.
    * **트레일러 (Trailer)**: 데이터의 **뒤쪽**에 붙는 제어 정보 (주로 오류 검출용).

---

## 5. 네트워크 주소 체계와 식별자
* **IP 주소 (IP Address)**: 인터넷에 연결된 호스트와 라우터를 식별하는 고유한 논리적 주소.
* **포트 번호 (Port Number)**: 인터넷 통신의 최종 종착점(운영체제 내의 특정 프로세스/프로그램)을 나타내는 식별자.
    * *아키텍트 노트*: 하나의 공인 IP는 여러 컴퓨터가 공유(NAT)할 수 있지만, **단일 호스트(PC) 내에서 특정 포트는 하나의 프로그램만 점유**할 수 있음.
* **도메인 이름 (Domain Name)**: 사람이 기억하기 쉬운 IP 주소의 별명 (예: `www.google.com`).
    * 실제 통신을 위해서는 DNS(Domain Name System)를 통해 반드시 **IP 주소로 변환**해야 함.

---

## 6. 클라이언트-서버 (Client/Server) 모델
* **정의**: 두 프로그램이 상호 작용하는 분산 애플리케이션 아키텍처 방식.
* **역할 분담**:
    * **클라이언트 (Client)**: 서비스를 **요청(Request)**하는 쪽.
    * **서버 (Server)**: 클라이언트의 요청을 대기하다가, 요청이 오면 해당 서비스를 **처리(Process)**하고 응답(Response)하는 쪽.

# [Study] 소켓(Socket) 프로그래밍 기초 및 OS별 특징

## 7. 소켓 (Socket)의 기본 개념
* **데이터 타입으로서의 소켓**:
    * 리눅스/유닉스의 **파일 디스크립터(File Descriptor)** 혹은 윈도우의 **핸들(Handle)**과 유사한 개념의 데이터 타입.
    * 생성 및 설정 과정이 끝나면, 운영체제(OS)의 통신 관련 정보를 참조하여 네트워크 데이터 송수신 등의 작업을 편리하게 수행할 수 있음.
* **통신 종단점 (Terminal / Endpoint)**:
    * 응용 프로그램(Application)은 하부의 복잡한 네트워크 라우팅을 신경 쓰지 않고, **자신의 소켓이 상대편의 소켓과 논리적으로 직접 연결된 것으로 간주**하고 데이터를 주고받음.
* **네트워크 프로그래밍 인터페이스**:
    * TCP/IP 프로토콜 스택에서 **응용 계층**과 **전송 계층** 사이에 위치하는 API로 간주함.
    * 통신 양단(Client-Server)이 모두 소켓 API를 사용할 필요는 없음 (TCP/IP 표준 프로토콜만 준수하면 이기종 간 통신 가능).

## 8. 윈도우 소켓 (Winsock) vs 유닉스/리눅스 소켓

### 8.1 윈도우 소켓의 기원
* 버클리 유닉스(Berkeley Unix)에서 개발한 네트워크 프로그래밍 인터페이스(BSD Sockets)를 윈도우 환경에서 사용할 수 있도록 포팅(Porting)한 것.

### 8.2 운영체제별 핵심 차이점
| 구분 | 윈도우 소켓 (Winsock) | 유닉스/리눅스 소켓 (Unix Socket) |
| :--- | :--- | :--- |
| **제공 방식** | **DLL (Dynamic-Link Library)**을 통해 대부분의 기능이 제공됨. | 운영체제 커널의 파일 시스템에 통합되어 제공됨. |
| **초기화/종료** | DLL 로드 및 해제를 위해 **초기화(`WSAStartup`)**와 **종료(`WSACleanup`)** 함수가 반드시 필요함. | 별도의 라이브러리 초기화 불필요. |
| **이벤트 처리** | 윈도우 특유의 GUI 및 **메시지 구동 방식(Message-driven)**으로 동작하므로, 이를 위한 확장 함수(`WSAAsyncSelect` 등)가 존재함. | `select`, `epoll` 등의 I/O 다중화 기법을 주로 사용. |
| **스레드 환경** | 운영체제 차원에서 멀티스레드를 지원하므로, 멀티스레드 환경에서 안정적으로 동작하는 구조와 전용 함수들이 존재함. | `pthread` 등 POSIX 스레드 표준을 주로 사용. |

---

## 9. 소켓 프로그래밍 오류 처리 (Error Handling)
> *안전한 코드를 작성하기 위해 반드시 숙지해야 하는 3가지 예외 처리 유형*

1.  **오류를 처리할 필요가 없는 경우**: 리턴값이 아예 없거나(`void`), 호출 시 항상 성공하는 일부 단순 소켓 함수들.
2.  **리턴값만으로 오류를 확인하는 경우**: 대표적으로 **`WSAStartup()`** 함수. 실패 시 직접 에러 코드를 리턴함.
3.  **리턴값 확인 후, 구체적 오류 코드를 별도로 확인하는 경우**: **대부분의 소켓 함수**. 리턴값으로 오류 발생 여부(`-1` 또는 `SOCKET_ERROR`)를 확인하고, 구체적인 원인은 에러 코드 함수(Win: `WSAGetLastError()`, Linux: `errno`)를 호출하여 파악함.


## 10. 소켓 응용 프로그램의 공통 생명주기 (Lifecycle) 

text
[시작] 윈속 초기화 (※ 윈도우에서만 필요: WSAStartup)
  ↓
[생성] 소켓 생성 (socket)
  ↓
[통신] 네트워크 통신 (bind, listen, accept / connect / send, recv)
  ↓
[종료] 소켓 닫기 (close / closesocket)
  ↓
[끝] 윈속 종료 (※ 윈도우에서만 필요: WSACleanup)
