# [Study] 멀티스레드와 다중 클라이언트 처리 (Windows 기준)

## 24. 다중 클라이언트 처리 문제 해결책
> *서버가 여러 클라이언트의 요청을 동시에 처리하기 위한 아키텍처 모델*

### 1. 스레드 기반 독립 처리 (Thread per Client)
* **방식**: 각 클라이언트가 접속할 때마다 새로운 스레드를 생성하여 독립적으로 처리함.
* **장점**: 흐름이 직관적이므로 소켓 입출력(I/O) 모델에 비해 **구현이 매우 쉽다**.
* **단점**: 클라이언트 수만큼 스레드가 생성되므로 컨텍스트 스위칭(Context Switching) 비용이 증가하고 **서버의 시스템 자원을 많이 사용**함. (대용량 서버에 부적합)

### 2. 소켓 입출력 모델 사용 (I/O Multiplexing / IOCP 등)
* **방식**: 소수의 스레드 풀(Thread Pool)을 미리 만들어 두고, 발생하는 I/O 이벤트만 비동기적으로 분배하여 처리함.
* **장점**: 소수의 스레드로 다수의 클라이언트를 처리하므로 **상대적으로 적은 시스템 자원을 사용**함. (대용량 서버의 표준)
* **단점**: 비동기 로직과 상태 관리가 들어가므로 **구현이 매우 어려움**.

## 25. 프로세스 (Process) vs 스레드 (Thread) 

| 구분 | 개념 및 특징 | 윈도우 운영체제에서의 관점 |
| :--- | :--- | :--- |
| **프로세스** | 운영체제로부터 CPU 시간과 자원을 할당받아 실행 중인 프로그램. | 코드, 데이터, 리소스를 메모리 영역에 담고 있는 일종의 **컨테이너(Container)** 개념. |
| **스레드** | 프로세스 내에서 생성되는 실제 실행 흐름. | CPU 시간을 실제로 할당받아 프로세스 메모리 영역의 코드를 수행하고 데이터를 조작하는 **실행 주체**. |

* *(참고: 일반적인 운영체제의 프로세스 개념 = 윈도우의 프로세스(컨테이너) + 최소 1개 이상의 메인 스레드)*

## 26. 스레드의 생성과 Win32 기본 자료형
* **스레드 생성에 필요한 핵심 요소**:
    1. **함수의 시작 주소**: C/C++ 프로그램에서는 '함수 이름' 자체가 해당 함수가 메모리에 올라간 시작 주소(포인터)를 의미함.
    2. **스레드 함수**: `f()` 함수와 같이 운영체제가 스레드를 생성한 직후 최초로 실행을 시작하는 진입점(Entry Point).

* **Win32 API 주요 자료형 (참고)**:
    * `LP`: Long Pointer (포인터형)
    * `DWORD`: Double Word (32비트 부호 없는 정수, 보통 크기나 개수를 나타냄)
    * `boolean`: 참/거짓 (Yes / No)을 나타내는 논리형.

## 27. 스레드 종료 방법 (Thread Termination)
> *아키텍트의 경고: 3번과 4번 방식은 리소스(메모리, 핸들) 누수를 유발하므로 실무에서는 절대 권장하지 않습니다.*

1. **스레드 함수가 리턴 (가장 안전하고 권장되는 방법)**: 할 일을 모두 마치고 자연스럽게 종료.
2. **`ExitThread()` 함수 호출**: 스레드 함수 내부에서 스스로 실행을 중단하고 종료함.
3. **`TerminateThread()` 함수 호출 (강제 종료)**: 다른 스레드가 외부에서 타겟 스레드를 강제로 죽임. (자원 반환이 보장되지 않아 매우 위험함).
4. **메인 스레드(프로세스) 종료**: `main()` 함수가 종료되면 프로세스 내의 다른 모든 워커 스레드(자식 스레드)도 자비 없이 강제 종료됨.

## 28. 스레드 스케줄링 및 우선순위 (Priority)
* **CPU / 스레드 스케줄링**: 윈도우 운영체제가 한정된 CPU 시간을 각 스레드에 적절히 분배하기 위한 정책.
* **우선순위 결정 방식**: 윈도우는 **[프로세스의 우선순위 클래스]**와 **[스레드의 우선순위 레벨]**을 조합하여 최종 스레드 우선순위(0~31)를 결정함.

### 28.1 프로세스 우선순위 클래스 (Process Priority Class)
> *프로세스 전체에 부여되는 베이스 우선순위*
1. `REALTIME_PRIORITY_CLASS` (실시간 - 가장 높음)
2. `HIGH_PRIORITY_CLASS` (높음)
3. `ABOVE_NORMAL_PRIORITY_CLASS` (높은 우선순위 / 윈도우 2000 이상)
4. `NORMAL_PRIORITY_CLASS` (보통 - 기본값)
5. `BELOW_NORMAL_PRIORITY_CLASS` (낮은 우선순위 / 윈도우 2000 이상)
6. `IDLE_PRIORITY_CLASS` (낮음)

### 28.2 스레드 우선순위 레벨 (Thread Priority Level)
> *프로세스 클래스 내에서 해당 스레드가 가지는 상대적 레벨*
1. `THREAD_PRIORITY_TIME_CRITICAL` (최상위)
2. `THREAD_PRIORITY_HIGHEST`
3. `THREAD_PRIORITY_ABOVE_NORMAL`
4. `THREAD_PRIORITY_NORMAL` (기본값)
5. `THREAD_PRIORITY_BELOW_NORMAL`
6. `THREAD_PRIORITY_LOWEST`
7. `THREAD_PRIORITY_IDLE` (최하위)

### 28.3 관련 API 함수
* **`SetThreadPriority(handle, level)`**: 특정 스레드의 우선순위 레벨을 변경할 때 사용.
* **`GetThreadPriority(handle)`**: 현재 설정된 스레드의 우선순위 레벨을 읽어올 때 사용.
