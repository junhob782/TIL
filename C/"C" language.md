메모리 구조
스택(Stack): 함수 호출 구조와 동일하게 작동 (지역변수, 매개변수). LIFO 구조.
힙(Heap): 사용자가 직접 관리하는 동적 할당 영역.

주소(Address)의 이해
정의: 변수가 할당된 메모리의 시작 주소.

할당 단위: 32bit/64bit 시스템 상관없이 1칸당 1byte씩 할당됨.
크기: int형은 4byte(4칸)를 차지하지만, 주소값 자체의 크기는 포인터 크기(시스템 비트 수)에 따라 고정됨.
핵심: 가리키는 데이터가 1byte(char)든 8byte(double)든, 주소값(포인터)의 크기는 동일하다.

엔디안 (Endianness)
리틀 엔디안 (Little Endian): 낮은 자릿수(LSB)부터 메모리에 저장 (인텔 계열).
빅 엔디안 (Big Endian): 높은 자릿수(MSB)부터 메모리에 저장 (네트워크 전송 등).

4. C언어 핵심: 포인터와 입출력
포인터 문법 (필수 암기)
pa = &a: 주소 구하기 (변수 a의 위치를 pa에 저장)
*pa = 10: 값 접근하기 (주소 pa로 가서 10을 대입)
*pa: 주소에 저장된 실제 값(Value).

함수와 스코프
재귀 함수: 자기 자신을 호출하는 함수 (스택 메모리 사용).
변수 접근: 가까운 블록(함수)의 변수부터 우선 접근한다 (Local scope 우선).

입출력 함수 (scanf)
값을 입력받을 변수의 **주소(&)**를 넘겨줘야 한다.
이미 주소인 경우(배열 이름, 포인터 등)에는 & 연산자가 필요 없다.

서식 지정자 및 특수문자 (수정본)
\n: 개행 문자 (줄바꿈)
%d: 10진수 정수 (Decimal) (※ 메모의 문자/숫자 반대 수정)
%c: 문자 한 개 (Char)
%f: 실수 (Float)
%s: 문자열 (String)

5. 자료구조와 알고리즘
 (배열)다차원 배열의 순회
메모리 효율을 위해 바깥쪽부터 순회
(행) $\rightarrow$ 안쪽(열) 순서로 도는 것이 정석.
예: int score[3][4]인 경우, [3](행)이 천천히 변하고 [4](열)가 빨리 변하도록 for문을 구성.
영상/그래픽 좌표계수학좌표 $(x, y)$와 달리 프로그래밍(행렬)에서는 y(행/세로)를 먼저 쓰고,
x(열/가로)를 뒤에 쓰는 경우가 많음. (arr[y][x])
