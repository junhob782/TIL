# [Study] Java 백엔드(AOP)와 로보틱스 산업 동향

## 55. Java 기초와 AOP (관점 지향 프로그래밍)
* **객체지향 프로그래밍 (OOP)**: 상태와 행위를 가진 '객체' 단위로 나누어 개발하는 Java의 기본 철학. 하지만 프로젝트가 커지면 중복 코드가 발생함.
* **관점 지향 프로그래밍 (AOP - Aspect-Oriented Programming)**: OOP를 보완하는 기법. 핵심 비즈니스 로직과 시스템 전반에 걸친 공통 부가 기능을 분리하여 관리함.
    * **횡단 관심사 (Cross-cutting Concerns)**: 시스템 전반에 걸쳐 흩어져 있는 공통 기능 (예: 메서드 호출 시간 로깅, 데이터 암호화/복호화, 트랜잭션 관리 등).
    * **애스펙트 (Aspect)**: 분리해낸 횡단 관심사들을 모듈화한 단위 (공통 기능들의 묶음).

## 56. Spring Framework 생태계와 백엔드 설계 
* **위빙 (Weaving)**: Spring 프레임워크(Spring.io)가 제공하는 핵심 기능 중 하나. 런타임이나 컴파일 타임에 `Aspect`(부가 기능)를 원본 핵심 코드에 **끼워 넣는(삽입하는) 과정**을 의미함. (원본 코드를 수정하지 않고도 부가 기능을 적용할 수 있게 해줌).
* **디자인 패턴 (Design Patterns)**: 소프트웨어 설계 시 자주 발생하는 문제들을 해결하기 위한 모범 답안(재사용 가능한 객체지향 설계 템플릿). Spring 내부도 팩토리(Factory), 싱글톤(Singleton), 프록시(Proxy) 패턴 등으로 이루어져 있음.
* **SQL과 데이터 모델링 (Modeling)**: 비즈니스 요구사항에 맞춰 데이터베이스의 테이블 구조와 관계(ERD)를 설계하는 과정.

## 57. 로봇 (ROBOT) 산업 분류 및 주요 기업 동향
> *소프트웨어(AI/제어)와 하드웨어(기구부)가 결합하는 최전선*

### 1. 휴머노이드 로봇 (Humanoid)
* **특징**: 인간과 유사한 형태를 가진 미래 지향적 로봇. 이족 보행 및 복잡한 균형 제어 기술이 필요함.
* **주요 기업 및 기술**:
    * **Boston Dynamics (보스턴 다이내믹스)**: 아틀라스(Atlas) 등 세계 최고 수준의 동적 제어 기술 보유.
    * **레인보우 로보틱스**: 카이스트 휴보(HUBO) 연구진이 창업한 국내 대표 휴머노이드 기업.
    * **로보티스 (ROBOTIS)**: 로봇 전용 액추에이터(다이나믹셀) 및 자율주행 로봇 제조.
    * > *아키텍트 노트: 최신 로봇 제어는 **ROS2 (Robot Operating System 2)**를 기반으로 합니다. 이는 스마트폰의 **Android OS**처럼, 다양한 제조사의 모터와 센서 등 하드웨어를 추상화하여 개발자가 소프트웨어(앱) 단위로 로봇을 제어할 수 있게 해주는 혁신적인 플랫폼입니다.*

### 2. 산업용 로봇 (Industrial Robot)
* **특징**: 공장 조립 라인 등에서 사용되며 거대하고, 무겁고, 비쌈. 작업자의 안전을 위해 반드시 펜스(Fence)를 치고 격리된 상태에서 매우 빠른 속도로 반복 작업을 수행함.
* **주요 기업**: 현대 로보틱스 (국내 산업용 로봇 점유율 1위) 등.

### 3. 협동 로봇 (Cobot - Collaborative Robot)
* **특징**: 휴머노이드와 기존 산업용 로봇의 중간 형태. 인간과 **같은 작업 공간에서 펜스 없이 상호작용(협동)**하며 일할 수 있도록 안전하게 설계된 신기술.
* **핵심 요구 기술**: 충돌을 감지하고 멈추는 기계전자(Mechatronics) 기술, AI 시각 처리, 그리고 모터를 정밀하게 제어하는 **펌웨어(Firmware)** 프로그래밍 역량이 필수적임.

## 58. 자바 가상 머신 (JVM: Java Virtual Machine) 
* **개요**: 자바 프로그램이 운영체제(OS)나 하드웨어에 종속되지 않고 실행될 수 있도록 도와주는 중간 실행 엔진.
* **스마트폰 플랫폼 (Android)**: 안드로이드 역시 다양한 제조사의 스마트폰 하드웨어 파편화를 극복하기 위해 가상 머신 기술(과거 Dalvik, 현재 ART)을 채택하여 앱(App)을 구동함.

### 58.1 가상 머신(VM) 도입의 장단점
* **장점 (WORA - Write Once, Run Anywhere)**: 하드웨어나 운영체제를 가리지 않고, 한 번 작성한 코드를 어디서나 동일하게 실행할 수 있음.
* **단점**:
    1. C/C++ 처럼 하드웨어에 맞춰 직접 빌드된 **최적화된 바이너리에 비해 실행 속도가 다소 느릴 수 있음** (중간 통역 과정이 필요하기 때문).
    2. 각 플랫폼(Windows, Linux, Mac 등)에 맞는 가상 머신(JRE/JDK)이 사전에 설치되어 있어야만 동작함.

## 59. 자바(Java)의 장단점 분석

### 59.1 자바의 강력한 장점
1. **객체지향 프로그래밍 (OOP)**: 간결하면서도 강력한 객체지향 설계(캡슐화, 상속, 다형성)를 지원하여 유지보수성이 뛰어남.
2. **개방형 구조 (Open Architecture)**: 여러 외부 기술 및 라이브러리와의 융합이 매우 용이함.
3. **플랫폼 독립성 (Platform Independent)**: 운영체제나 하드웨어 아키텍처가 바뀌어도 소스 코드를 다시 컴파일할 필요 없이 동일하게 실행됨.
4. **압도적인 생태계 (Ecosystem)**: Spring, Hibernate 등 수많은 검증된 오픈소스 프레임워크가 존재하여 개발 생산성을 극대화하고 유지보수 비용을 획기적으로 절감함.

### 59.2 자바의 한계와 단점
1. **극단적인 처리 속도 요구 분야**: 마이크로초(us) 단위의 성능이 중요한 시스템(예: OS 커널, 초고속 트레이딩 엔진)에는 적합하지 않을 수 있음. 
    * *(아키텍트 노트: 단, 일반적인 웹/백엔드 서버 환경에서는 JIT 컴파일러 덕분에 C/C++에 크게 뒤지지 않는 우수한 속도를 냅니다.)*
2. **GUI 애플리케이션 한계**: MS 윈도우 기반의 화려하고 빠릿빠릿한 데스크톱 응용 프로그램(GUI) 개발에는 C#이나 C++(MFC/Qt)에 비해 무겁고 적합하지 않음.
3. **하드웨어 직접 제어 불가**: 가상 머신(JVM)이라는 샌드박스 위에서 동작하므로, 메모리 주소(포인터)에 직접 접근하거나 하드웨어(센서, 액추에이터)를 정밀하고 민첩하게 조정해야 하는 펌웨어 프로그래밍에는 적합하지 않음.

## 60. 빅데이터(Big Data)와 자바의 관계 

### 60.1 빅데이터 개요
* 방대한 양의 데이터를 분석하여 의미 있는 정보(인사이트)를 추출하고, 이를 통해 사용자 맞춤형 서비스를 제공하거나 미래 현상을 예측하는 기술 (의학, 정치, 사회 등 전 분야 활용).

### 60.2 빅데이터 생태계의 표준, 자바
* 빅데이터를 수집하고 분석하는 핵심 소프트웨어(Hadoop, Apache Spark, Kafka, Elasticsearch 등)는 주로 **오픈소스 기반이며, 이들 생태계의 절대다수가 자바(Java) 또는 JVM 기반 언어(Scala 등)로 개발**되었음.
* 따라서 방대한 데이터 노드 간의 중계 및 분석 로직을 구현하고 커스터마이징하는 데 있어 자바는 대체 불가능한 필수 언어로 자리 잡고 있음.
