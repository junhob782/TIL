# [Study] 파이썬 데이터 과학의 핵심: 넘파이(NumPy)

## 1. 넘파이(NumPy) 개요와 배열의 특징
* **넘파이 (NumPy)**: 다차원 배열(Tensor)을 효율적으로 다루기 위해 파이썬 생태계에 제공되는 가장 기본적이고 강력한 데이터 과학 라이브러리임.
* **파이썬 리스트와의 근본적 차이점**:
    * **단일 자료형**: 리스트는 여러 자료형을 섞어 쓸 수 있지만 메모리 접근 속도가 느림. 반면, 넘파이 배열(`ndarray`)은 동일한 자료형만 저장하여 각 항목의 저장 공간이 일정하므로 데이터 접근 및 연산 속도가 압도적으로 빠름.
    * **벡터 연산**: 리스트 간의 덧셈(`+`)은 요소들이 단순히 뒤로 연결(Concatenation)되지만, 넘파이는 데이터를 선형대수의 벡터(Vector)로 간주하여 같은 인덱스끼리 더하고 빼는 산술 연산을 수행함.

---

## 2. 다차원 배열 (`ndarray`)의 주요 속성 
> *프로그램의 오류를 찾거나 배열의 정보를 손쉽게 조회할 때 사용하는 속성들*

| 속성 (Attribute) | 설명 및 특징 |
| :--- | :--- |
| **`ndim`** | 배열의 축(Axis) 또는 차원의 개수를 나타냄. |
| **`shape`** | 배열의 형태(형상)를 `(행, 열)` 형태의 튜플로 반환함. 가장 자주 쓰이는 속성. |
| **`size`** | 배열이 포함하고 있는 전체 원소의 개수 (예: 2x3 배열의 size는 6). |
| **`dtype`** | 배열 원소의 자료형 (예: `int64`, `float64`). |

---

## 3. 벡터화 연산과 브로드캐스팅 (Broadcasting) 
* **벡터화 연산 (Vectorized Operation)**: 파이썬의 `for` 반복문 없이, 하나의 명령을 여러 데이터에 동시에 적용하여 병렬적으로 연산하는 최적화 기법.
* **브로드캐스팅 (Broadcasting)**: 차원이나 크기가 서로 다른 배열 간에 연산을 수행할 때, 작은 배열이나 스칼라값을 큰 배열의 형상(Shape)에 맞게 자동으로 쫙 늘려(확장시켜) 계산을 가능하게 해주는 강력한 기능.

---

## 4. 필수 다차원 배열 생성 및 변환 함수

### 4.1 초기화 및 수열 생성
1. **`np.zeros((n, m))` / `np.ones((n, m))`**: 모든 값을 `0` 또는 `1`로 초기화한 배열 생성.
2. **`np.arange(start, stop, step)`**: 파이썬의 `range`처럼 지정된 간격(`step`)으로 연속적인 수열의 배열 생성.
3. **`np.linspace(start, stop, num)`**: 지정된 구간을 동일한 간격으로 쪼개어 정확히 `num` 개수만큼의 데이터를 생성함.

### 4.2 형태 변경 (`reshape`)과 평탄화 (`flatten`)
* **`reshape(n, m)`**: 데이터 개수(size)는 그대로 유지한 채 배열의 차원과 형태를 변경함. 인수로 `-1`을 넣으면 남은 원소 개수에 맞춰 나머지 차원이 자동으로 계산됨.
* **`flatten()`**: 2차원 이상의 다차원 배열을 1차원 배열로 쭉 펴주는(평탄화) 함수.

---

## 5. 넘파이 스타일 슬라이싱과 논리 인덱싱
* **넘파이 슬라이싱 (`a[행, 열]`)**: 쉼표(`,`)를 기준으로 각 축(Axis)에 별도의 슬라이싱을 적용하여, 큰 데이터에서 특정한 행과 열만 추려내어 뽑아내는 데 매우 유용함.
* **논리 인덱싱 (Logical / Boolean Indexing)**: 특정한 조건식(예: `a % 2 == 0`)을 배열에 주어 불리언(`True`/`False`) 배열을 만든 뒤, 조건에 부합하는(`True`) 원소만 추출해 내는 효율적인 기법.

---

## 6. 리덕션(Reduction)과 기초 통계 연산
> *여러 개의 수로 이루어진 배열을 하나의 수로 집계(Aggregation)하는 기능*

* **리덕션 함수**: `sum()`, `mean()`, `min()`, `max()` 등을 사용하며, 특정 축(`axis=0` 또는 `axis=1`)을 지정해 행렬의 가로/세로 방향 집계가 가능함.
* **통계 지표**:
    * **분산 (`np.var`)**: 데이터가 평균값으로부터 얼마나 멀리 퍼져 있는지를 정량화한 척도 (편차 제곱의 평균).
    * **표준 편차 (`np.std`)**: 분산에 제곱근을 씌워 실제 값과 근사시킨 척도.

---

## 7. 가상 데이터 시뮬레이션: 난수 (Random)
* **의사 난수 (Pseudo Random)**: 컴퓨터 알고리즘을 통해 생성되어 진정한 난수와 매우 유사한 특징을 가지는 수.
* **시드 (`seed`)**: 난수 생성 알고리즘의 최초 시작값. 시드값을 특정 숫자로 고정하면 매번 완벽히 동일한 난수의 열을 얻을 수 있음.
* **주요 난수 함수**:
    * `rand()`: 균등 분포 난수 추출.
    * `randn()`: 평균 0, 분산 1인 표준 정규 분포 난수 추출.
    * `randint(최소, 최대)`: 지정 범위 내의 정수 난수 생성.

---

## 8. [아키텍트 심화] 넘파이는 왜 압도적으로 빠른가? (SIMD) 
* **배열 프로그래밍 (Array Programming)**: 데이터를 구성하는 값 전체에 대해 한 번에 연산이 적용되는 패러다임. 전통적인 `for` 반복문(스칼라 언어)을 대체함.
* **데이터 병렬성 (Data Parallelism)**: 처리해야 하는 데이터를 잘게 쪼개어 동일한 작업을 여러 코어가 동시에 진행하는 방식.
* **SIMD (Single Instruction Multiple Data)**: CPU가 스레드를 새로 생성하지 않고도 하나의 연산 명령을 다수의 데이터에 동시에 적용하는 하드웨어 아키텍처. 넘파이는 이를 내부적으로 적극 활용하여 명시적 반복문 대비 수십 배에서 수백 배 이상의 극한 성능 향상을 이끌어 냄.
