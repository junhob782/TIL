## 80. 파이썬 (Python) 개요 및 특징 
* **장점**:
    1. **높은 생산성**: 문법이 영어 문장처럼 매우 간단하고 직관적이어서 배우기 쉽고 개발 속도가 빠름.
    2. **방대한 생태계**: 웹 서버(Django, FastAPI), 해킹 도구, IoT, 데이터 분석, 인공지능(AI) 등 거의 모든 분야에 활용할 수 있는 풍부한 라이브러리를 갖춤.
* **단점**:
    1. **환경 파편화**: 윈도우, 맥, 리눅스 등 운영체제마다 환경 설정 및 경로 사용 방법이 조금씩 달라 초기 세팅에 주의가 필요함.
    2. **느린 실행 속도**: C/C++ 같은 컴파일 언어에 비해 연산 속도가 느림.
* **구동 원리 (왜 느린가?)**: 
    * 파이썬은 작성된 텍스트 코드(`.py`)를 파이썬 가상 머신(PVM) 위에서 한 줄씩 실시간으로 번역(Interpret)하며 실행함. 이 번역기(인터프리터, 주로 CPython)가 C언어로 작성되어 동작하는 과정에서 오버헤드가 발생하기 때문에 근본적으로 느림.

## 81. Python vs PyTorch vs PyCharm 완벽 비교
> *파이썬 생태계의 3가지 핵심 요소를 명확히 구분해야 합니다.*

| 구분 | 정의 | 역할 및 특징 |
| :--- | :--- | :--- |
| **Python** | **프로그래밍 언어 (Language)** | 컴퓨터에게 명령을 내리기 위한 기본 문법이자 규칙. (예: `if`, `for`, `print`). 모든 AI와 웹 개발의 뼈대가 되는 범용 언어. |
| **PyTorch** | **머신러닝 전용 프레임워크 (Framework)** | 파이썬 언어로 만들어진(내부 연산은 C/C++로 최적화된) **인공지능 도구 상자**. 메타(Meta)에서 개발.<br>- *역할*: 신경망 공식(예: $Y = W \cdot X + b$)이나 역전파, GPU 연산 등 복잡한 수학/AI 코드를 파이썬 위에서 쉽게 조립할 수 있게 해주는 특수 목적 부품. |
| **PyCharm** | **통합 개발 환경 (IDE)** | 파이썬 코드를 쉽고 편하게 작성할 수 있도록 도와주는 **전용 텍스트 에디터(프로그램)**. JetBrains에서 개발.<br>- *역할*: 자동 완성, 코드 오류 하이라이팅, 디버깅 기능 등을 제공하여 개발자의 생산성을 극대화해 줌. |

## 82. 파이썬 코드의 기본 구성 요소
1. **표현식 (Expression)**: 파이썬에서 어떠한 '값'을 만들어내는 간단한 코드. (예: `273`, `10 + 20`, `"Hello"`)
2. **문장 (Statement)**: 표현식이 하나 이상 모여서 **실행 가능한 하나의 동작**을 구성한 것. (그 자체로 어떠한 결과를 만들어내지 못하면 문장이 아님).
3. **프로그램 (Program)**: 여러 개의 문장이 모여 논리적인 흐름을 형성한 것.
4. **리터럴 (Literal)**: 변수에 할당되는, 코드 상에 직접 입력된 '어떠한 값 자체' (자료).
5. **연산자 (Operator)**: 스스로 값이 되는 것이 아니라, 값(리터럴)들 사이에 들어가 계산 등 무언가 연산을 지시하는 기호. (예: `+`, `-`, `*`)
6. **주석 (Comment)**: 프로그램의 실제 실행 진행에는 전혀 영향을 주지 않는 코드. 주로 개발자가 프로그램 코드에 대한 설명이나 메모를 남기기 위해 사용함 (`#` 기호 사용).

## 83. 식별자와 네이밍 컨벤션 (Naming Convention)
* **키워드 (Keyword)**: 파이썬에서 특별한 의미로 사용하기로 미리 예약해 놓은 단어 (예: `True`, `False`, `for`, `if`). 식별자로 사용할 수 없음.
* **식별자 (Identifier)**: 프로그래밍 언어에서 변수나 함수, 클래스 등에 **이름을 붙일 때 사용하는 단어**.
    * **규칙**: 키워드 사용 불가, 숫자로 시작 불가, 공백 포함 불가, 특수문자는 언더바(`_`)만 허용됨. 의미가 명확한 알파벳 단어로 짓는 것이 관례임.

### 83.1 대표적인 표기법 (Naming Case)
* **스네이크 케이스 (Snake Case)**: 단어들 사이에 언더바(`_`)를 붙여서 뱀처럼 길게 이어 쓰는 방식. 주로 **변수명, 함수명**에 사용함. (예: `login_user_name`)
* **캐멀 케이스 (Camel Case / Pascal Case)**: 단어들의 첫 글자를 대문자로 만들고 언더바 없이 이어 쓰는 방식. 낙타의 혹처럼 보임. 파이썬에서는 주로 **클래스명**에 사용함. (예: `LoginUserName`)

## 84. 함수(Function)와 메서드(Method), 생성자(Constructor)
* **함수 (Function)**: 특정 객체나 클래스와 상관없이 전역적으로 혼자 독립해서 존재할 수 있는 기능.
* **메서드 (Method)**: 특정 객체(클래스) 안에 종속되어 들어있는 함수. 
    * > **아키텍트 노트**: 자바의 `this`와 달리, 파이썬에서 클래스 내부의 메서드는 첫 번째 매개변수로 반드시 **`self`**를 가져야 합니다. `self`는 생성자가 아니라 '자기 자신 객체'를 가리키는 참조자입니다.
* **생성자 (Constructor)**: 클래스(설계도)를 바탕으로 객체가 힙 메모리에 올라갈 때, 제일 먼저 자동으로 호출되어 변수들을 초기화해 주는 특별한 메서드 (파이썬에서는 `__init__` 메서드 사용). 직접 만들지 않으면 아무 일도 하지 않는 '디폴트 생성자'가 자동으로 동작함.

## 85. 파이썬의 객체지향 프로그래밍 (OOP) 흐름
* 파이썬은 내부의 거의 모든 데이터가 객체(Object)로 이루어진 언어입니다.
* **동작 흐름**: `프로그램` $\rightarrow$ `O.O.P (객체지향 프로그래밍)` $\rightarrow$ Base: `클래스(Class)` $\rightarrow$ `인스턴스(Instance)` 생성 $\rightarrow$ `캡슐화 / 상속 / 다형성` 특징 활용.

## 86. 자료(Data)와 출력 함수
* **자료 (Data)**: 프로그램이 메모리에 올려서 연산하고 처리할 수 있는 모든 것을 의미함.
* **`print()` 함수**: 파이썬의 표준 출력 함수. 출력하고 싶은 자료나 변수들을 괄호 `()` 안에 나열하여 콘솔 화면에 보여줌.

## 87. 파이썬의 3대 기본 자료형 (Data Types)
> *파이썬이 메모리에 데이터를 저장하고 취급하는 기본 형태*

| 자료형 | 영문명 및 클래스 | 의미 및 활용 예시 |
| :--- | :--- | :--- |
| **문자열** | String (`str`) | 메시지 내용, 메일 제목 등 글자들의 집합. (예: `"안녕하세요"`) |
| **숫자** | Number (`int`, `float`) | 물건의 가격, 성적, 센서값 등 계산이 가능한 수치. (예: `273`, `3.14`) |
| **불** | Boolean (`bool`) | 참(`True`)과 거짓(`False`) 두 가지 값만 가짐. 친구의 로그인 상태, 스위치 On/Off 등 상태 판별에 사용. |

### 87.1 자료형 확인 함수: `type()`
* 현재 다루고 있는 값이나 변수가 어떤 자료형(클래스)인지 확인하고 싶을 때 사용하는 내장 함수.
* **사용 예시**: `print(type("안녕하세요"))` $\rightarrow$ 실행 결과로 `<class 'str'>`이 출력됨.

## 88. [핵심] 문자열의 불변성 (Immutability)
* **특징**: 파이썬에서 한 번 메모리에 생성된 문자열은 **임의로 부분 수정이 절대 불가능**함.
* **해결책**: 원본 문자열의 일부를 수정하는 대신, 가공을 거쳐 **새로운 문자열 객체를 만들어** 새로 할당(작성)해야 함.
* > **아키텍트 노트**: C언어의 문자 배열처럼 `text[0] = 'A'` 방식으로 글자 하나만 쏙 바꾸려고 시도하면 파이썬은 즉시 에러(`TypeError`)를 발생시킵니다. 메모리 안정성을 위한 파이썬의 매우 중요한 특징입니다.

## 89. 독스트링 (DocString)과 멀티라인 문자열
* **형태**: 큰따옴표 3개 `"""` 또는 작은따옴표 3개 `'''`로 문자열의 위아래를 감싸서 표현함.
* **용도 및 특징**:
    1. **멀티라인 출력**: 코드 내부에 엔터(줄바꿈)를 쳐서 작성한 형태(WYSIWYG) 그대로 화면에 똑같이 출력해 줌. (별도의 `\n` 제어 문자가 필요 없음).
    2. **서류형 문자열 (DocString)**: 클래스나 함수가 시작되는 바로 밑에 작성하여, 해당 코드의 목적, 파라미터, 반환값 등을 설명하는 **공식 설명서(메뉴얼)** 역할을 함.

## 90. 문자열 (String) 개요와 인덱스 (Index)
* **문자열 (String)**: 하나 또는 여러 개의 문자로 구성된 기본 데이터 형.
* **인덱스 (Index)**: 데이터 구조 내에서 요소의 '위치'를 나타내는 번호표.
    1. **제로 인덱스 (Zero Index)**: 숫자를 **0**부터 셈. *(파이썬, C, 자바 등 대부분의 언어가 채택)*
    2. **원 인덱스 (One Index)**: 숫자를 **1**부터 셈. *(R, MATLAB 등 일부 통계/수학 언어에서 사용)*
* > **아키텍트 팁 (음수 인덱싱)**: 파이썬은 뒤에서부터 요소를 찾을 때 사용하는 **음수 인덱스**를 지원합니다. 맨 마지막 문자는 `-1`, 그 앞은 `-2`가 됩니다. (예: `text[-1]`)

---

## 91. 인덱싱(Indexing)과 슬라이싱(Slicing) 
> *문자열 내부의 데이터를 추출하는 파이썬의 가장 강력하고 편리한 무기*

### 91.1 인덱싱 (문자 선택 연산자: `[]`)
* **개념**: `[]` 기호를 이용해 문자열의 **특정 위치(인덱스)에 있는 문자 딱 1개**를 콕 집어 참조하는(가져오는) 것.
* **사용법**: 대괄호 안에 선택할 문자의 위치를 지정함. (예: `name[0]`)

### 91.2 슬라이싱 (부분 추출 연산자: `[:]`)
* **개념**: `[:]` 기호를 이용해 문자열의 **특정 구간(일부)을 통째로 잘라내어 추출**하는 것. 
* **사용법**: `문자열[시작위치:끝위치]` 형태로 사용함. (단, '끝 위치'의 문자는 포함되지 않고 그 앞까지만 잘림).
* **[핵심 특징] 원본 보존**: 슬라이싱을 통해 잘라낸 문자열은 완전히 새로운 문자열 객체로 반환되며, **원본 문자열 데이터는 절대 변하지 않음**. (문자열의 불변성).

## 92. 내장 함수와 예외 처리 (len & IndexError)

### 92.1 문자열 길이 확인: `len()` 함수
* **역할**: Length의 약자로, 문자열의 전체 길이를 구할 때 활용하는 파이썬 내장 함수.
* **사용법**: 괄호 내부에 문자열(또는 변수)을 넣으면 문자열을 구성하는 총 문자 개수를 정수로 반환해 줌. (예: `len("Hello")` $\rightarrow$ `5`)

### 92.2 `IndexError` 예외 (런타임 오류)
* **발생 원인**: 인덱싱을 할 때, 실제 문자열이나 리스트가 **가진 길이를 넘어서는(존재하지 않는) 요소나 글자의 인덱스 번호를 선택**할 경우 발생하는 에러.
* **방어 코딩**: 인덱스 번호를 변수로 사용할 때는 항상 `len()` 함수를 이용해 문자열의 실제 길이를 초과하지 않도록 검사하거나 제어해야 함.

## 93. 복소수 (Complex Number) 자료형
* **사용 이유**: 수학 연산, 특히 인공지능(AI), 신호 처리, 양자 컴퓨팅 등에서 **계산을 편리하고 직관적으로 하기 위해** 사용함.
* **파이썬의 특징**: 파이썬은 내부적으로 복소수를 기본 자료형으로 지원함. (단, 수학에서 허수부를 표현하는 `i` 대신, 파이썬 등 공학 분야에서는 전류 `i`와 헷갈리지 않도록 **`j`**를 사용함. 예: `3 + 4j`).
