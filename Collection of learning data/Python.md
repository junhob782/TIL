# [Study] 파이썬(Python) 기초와 인공지능 생태계

## 80. 파이썬 (Python) 개요 및 특징 
* **장점**:
    1. **높은 생산성**: 문법이 영어 문장처럼 매우 간단하고 직관적이어서 배우기 쉽고 개발 속도가 빠름.
    2. **방대한 생태계**: 웹 서버(Django, FastAPI), 해킹 도구, IoT, 데이터 분석, 인공지능(AI) 등 거의 모든 분야에 활용할 수 있는 풍부한 라이브러리를 갖춤.
* **단점**:
    1. **환경 파편화**: 윈도우, 맥, 리눅스 등 운영체제마다 환경 설정 및 경로 사용 방법이 조금씩 달라 초기 세팅에 주의가 필요함.
    2. **느린 실행 속도**: C/C++ 같은 컴파일 언어에 비해 연산 속도가 느림.
* **구동 원리 (왜 느린가?)**: 
    * 파이썬은 작성된 텍스트 코드(`.py`)를 파이썬 가상 머신(PVM) 위에서 한 줄씩 실시간으로 번역(Interpret)하며 실행함. 이 번역기(인터프리터, 주로 CPython)가 C언어로 작성되어 동작하는 과정에서 오버헤드가 발생하기 때문에 근본적으로 느림.

## 81. Python vs PyTorch vs PyCharm 완벽 비교
> *파이썬 생태계의 3가지 핵심 요소를 명확히 구분해야 합니다.*

| 구분 | 정의 | 역할 및 특징 |
| :--- | :--- | :--- |
| **Python** | **프로그래밍 언어 (Language)** | 컴퓨터에게 명령을 내리기 위한 기본 문법이자 규칙. (예: `if`, `for`, `print`). 모든 AI와 웹 개발의 뼈대가 되는 범용 언어. |
| **PyTorch** | **머신러닝 전용 프레임워크 (Framework)** | 파이썬 언어로 만들어진(내부 연산은 C/C++로 최적화된) **인공지능 도구 상자**. 메타(Meta)에서 개발.<br>- *역할*: 신경망 공식(예: $Y = W \cdot X + b$)이나 역전파, GPU 연산 등 복잡한 수학/AI 코드를 파이썬 위에서 쉽게 조립할 수 있게 해주는 특수 목적 부품. |
| **PyCharm** | **통합 개발 환경 (IDE)** | 파이썬 코드를 쉽고 편하게 작성할 수 있도록 도와주는 **전용 텍스트 에디터(프로그램)**. JetBrains에서 개발.<br>- *역할*: 자동 완성, 코드 오류 하이라이팅, 디버깅 기능 등을 제공하여 개발자의 생산성을 극대화해 줌. |

## 82. 파이썬 코드의 기본 구성 요소
1. **표현식 (Expression)**: 파이썬에서 어떠한 '값'을 만들어내는 간단한 코드. (예: `273`, `10 + 20`, `"Hello"`)
2. **문장 (Statement)**: 표현식이 하나 이상 모여서 **실행 가능한 하나의 동작**을 구성한 것. (그 자체로 어떠한 결과를 만들어내지 못하면 문장이 아님).
3. **프로그램 (Program)**: 여러 개의 문장이 모여 논리적인 흐름을 형성한 것.
4. **리터럴 (Literal)**: 변수에 할당되는, 코드 상에 직접 입력된 '어떠한 값 자체' (자료).
5. **연산자 (Operator)**: 스스로 값이 되는 것이 아니라, 값(리터럴)들 사이에 들어가 계산 등 무언가 연산을 지시하는 기호. (예: `+`, `-`, `*`)
6. **주석 (Comment)**: 프로그램의 실제 실행 진행에는 전혀 영향을 주지 않는 코드. 주로 개발자가 프로그램 코드에 대한 설명이나 메모를 남기기 위해 사용함 (`#` 기호 사용).

## 83. 식별자와 네이밍 컨벤션 (Naming Convention)
* **키워드 (Keyword)**: 파이썬에서 특별한 의미로 사용하기로 미리 예약해 놓은 단어 (예: `True`, `False`, `for`, `if`). 식별자로 사용할 수 없음.
* **식별자 (Identifier)**: 프로그래밍 언어에서 변수나 함수, 클래스 등에 **이름을 붙일 때 사용하는 단어**.
    * **규칙**: 키워드 사용 불가, 숫자로 시작 불가, 공백 포함 불가, 특수문자는 언더바(`_`)만 허용됨. 의미가 명확한 알파벳 단어로 짓는 것이 관례임.

### 83.1 대표적인 표기법 (Naming Case)
* **스네이크 케이스 (Snake Case)**: 단어들 사이에 언더바(`_`)를 붙여서 뱀처럼 길게 이어 쓰는 방식. 주로 **변수명, 함수명**에 사용함. (예: `login_user_name`)
* **캐멀 케이스 (Camel Case / Pascal Case)**: 단어들의 첫 글자를 대문자로 만들고 언더바 없이 이어 쓰는 방식. 낙타의 혹처럼 보임. 파이썬에서는 주로 **클래스명**에 사용함. (예: `LoginUserName`)

## 84. 함수(Function)와 메서드(Method), 생성자(Constructor)
* **함수 (Function)**: 특정 객체나 클래스와 상관없이 전역적으로 혼자 독립해서 존재할 수 있는 기능.
* **메서드 (Method)**: 특정 객체(클래스) 안에 종속되어 들어있는 함수. 
    * > **아키텍트 노트**: 자바의 `this`와 달리, 파이썬에서 클래스 내부의 메서드는 첫 번째 매개변수로 반드시 **`self`**를 가져야 합니다. `self`는 생성자가 아니라 '자기 자신 객체'를 가리키는 참조자입니다.
* **생성자 (Constructor)**: 클래스(설계도)를 바탕으로 객체가 힙 메모리에 올라갈 때, 제일 먼저 자동으로 호출되어 변수들을 초기화해 주는 특별한 메서드 (파이썬에서는 `__init__` 메서드 사용). 직접 만들지 않으면 아무 일도 하지 않는 '디폴트 생성자'가 자동으로 동작함.
