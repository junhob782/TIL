> [!IMPORTANT]
> **올바른 배치의 핵심 규칙**
> - 같은 대각선상의 두 휠은 **같은 타입** (위 그림에서 좌상-우하가 A, 우상-좌하가 B)
> - 좌우 옆쪽으로 인접한 두 휠은 **다른 타입**
>
> 원문의 *"섀시 조합은 ABAB"* 는 **좌측 두 휠이 A, 우측 두 휠이 B** 처럼 좌→우 순서로 읽었을 때의 표기로 해석할 수 있습니다. 다만 위 다이어그램의 대각선 규칙으로 이해하는 것이 더 직관적입니다.

🟢 **보충 — 왜 대각선 규칙인가**

옴니디렉셔널 이동을 위해서는 4개 휠의 추진력 벡터가 X축과 Y축 양쪽으로 적절히 분해되어 합산되어야 합니다. 같은 타입 휠끼리 대각선으로 배치하면 4개 추진력이 **회전 대칭성** 을 가져 임의 방향 이동이 수학적으로 가능해집니다.

## 1.4 전방향 운동의 원리

### 핵심 메커니즘

> [!NOTE]
> 메카넘 휠 섀시의 전방향 이동은 **"4개 휠의 회전 방향과 속도를 조합하여 지면에 작용하는 추진력 벡터의 합을 임의의 방향으로 만드는 것"** 으로 구현됩니다.

각 휠이 회전할 때:
1. 휠 자체는 차축 방향으로 회전
2. 지면과 접촉한 **롤러는 45°로 기울어져 있어** 추진력이 비스듬한 방향으로 작용
3. 4개 휠의 비스듬한 추진력이 **벡터 합산** 되어 차체의 최종 운동 방향 결정

### 기본 운동 패턴

| 목표 운동 | 4개 휠 회전 방향 |
| :--- | :--- |
| **전진** | 4개 휠 모두 전방 회전 |
| **후진** | 4개 휠 모두 후방 회전 |
| **우측 평행 이동** | 좌상 전방, 우상 후방, 좌하 후방, 우하 전방 |
| **좌측 평행 이동** | 위 패턴의 반대 |
| **제자리 시계 회전** | 좌측 두 휠 전방, 우측 두 휠 후방 |
| **제자리 반시계 회전** | 좌측 두 휠 후방, 우측 두 휠 전방 |
| **대각선 이동** | 두 휠만 회전 (대각선상의 두 휠) |

### 🟡 원문 표현 보정

> 원문: *"둘레가 작은 메카넘 휠은 측면으로 미끄러지는 능력이 있음"*

이 표현은 부정확합니다. **휠 둘레의 크기와 측면 이동 능력은 무관** 합니다. 정확한 의미는:

> ✅ **정정** : *"메카넘 휠은 휠 외주의 롤러들이 자유 회전(passive rotation)하므로, 차축 방향으로 능동 회전하는 동시에 롤러 축 방향으로 자유롭게 미끄러질 수 있다. 이 두 자유도의 조합이 측면 이동을 가능하게 한다."*

🟢 **보충 — 추진력 방향 분해**

각 롤러는 휠 회전 시:
- **롤러 축 방향** : 자유 회전이므로 마찰력 없음 (수동적으로 미끄러짐)
- **롤러 축 수직 방향** : 지면과 마찰 → 추진력 발생

롤러 축이 45°이므로, 추진력은 정확히 휠 진행 방향에서 45° 기울어진 방향으로 발생합니다. 이것이 메카넘 휠 특유의 거동을 만드는 원인입니다.

## 1.5 운동학적 모델의 매개변수

메카넘 휠 차체의 운동을 수학적으로 다루기 위해 정의되는 변수들입니다.

### 차체 속도 (몸체 좌표계 기준)

| 기호 | 의미 |
| :--- | :--- |
| **`Vx`** | 차체의 X축 속도 (앞뒤 방향, 전진이 양수) |
| **`Vy`** | 차체의 Y축 속도 (좌우 방향, 좌측 이동이 양수 — ROS 표준) |
| **`Vω`** (또는 `ω_z`) | 차체의 각속도 (반시계 회전이 양수) |

### 차체 기하 매개변수

| 기호 | 의미 |
| :--- | :--- |
| **`L`** | **좌우** 휠 중심 사이의 거리 (차체 폭 방향, **트랙(track)**) |
| **`H`** (또는 `2·l_x`) | **앞뒤** 휠 중심 사이의 거리 (차체 길이 방향, **휠베이스(wheelbase)**) |
| **`r`** | 휠 반지름 |
| **`θω`** | 차체 중심에서 우측 앞 휠 중심을 잇는 선과 X축이 이루는 각도 (정사각 배치 시 45°) |

### 휠 속도 (4개)

각 휠의 회전 각속도(`rad/s`)를 다음과 같이 표기합니다.

| 기호 | 위치 |
| :--- | :--- |
| **`A`** 또는 **`ω_fl`** (front-left) | 좌측 앞 휠 |
| **`B`** 또는 **`ω_fr`** (front-right) | 우측 앞 휠 |
| **`C`** 또는 **`ω_rl`** (rear-left) | 좌측 뒤 휠 |
| **`D`** 또는 **`ω_rr`** (rear-right) | 우측 뒤 휠 |

### 보조 변수 (오른쪽 앞바퀴 운동 분해)

| 기호 | 의미 |
| :--- | :--- |
| **`VBx`** | 오른쪽 앞바퀴 중심의 X 방향 속도 성분 |
| **`VBy`** | 오른쪽 앞바퀴 중심의 Y 방향 속도 성분 |

🟡 **참고** — 원문에서는 `VBx, VBy` 가 *"메카넘 휠의 X/Y축 속도"* 로 표기되어 있지만, 맥락상 **차체 중심이 아닌 특정 휠(우측 앞바퀴) 중심점의 속도** 로 해석됩니다. 변수명 첫 글자 `B` 가 이를 시사합니다 (오른쪽 = right = B).

## 1.6 운동학 공식 유도

### 핵심 가정 (단순화)

수학적 모델 단순화를 위해 다음 두 가지 이상적 가정을 도입합니다.

> [!IMPORTANT]
> **가정 ①** : 모든 바퀴는 지면에서 미끄러지지 않으며(no slip), 마찰력이 충분하다.
>
> **가정 ②** : 4개의 바퀴는 직사각형(또는 정사각형)의 모서리에 배치되어 있으며, 서로 평행하다.

🟢 **보충 — 왜 이런 가정이 필요한가**

- **무미끄럼 가정** : 실제로 메카넘 휠은 옴니 이동 시 일정량의 미끄럼이 필연적이지만 (특히 대각선 이동 시), 이를 무시하면 운동학식이 선형이 되어 분석이 쉬워집니다.
- **사각 배치 가정** : 비대칭 배치에서는 회전 시 회전 중심이 차체 기하 중심에서 벗어나는 보정항이 추가됩니다.

## 1.1 휠 유형 분류

이동 로봇의 휠은 크게 **세 가지 유형** 으로 나눌 수 있습니다.

### ① 메카넘 휠(Mecanum Wheel) 차량

> [!NOTE]
> 1973년 스웨덴 엔지니어 **Bengt Erland Ilon** 이 발명한 옴니디렉셔널(전방향) 휠입니다.

- 여러 개의 작고 자유롭게 회전하는 **롤러(roller)** 를 휠 외주에 부착
- 일반적으로 섀시 하단 4개 위치에 배치되며, 각 휠은 **독립적으로 모터 구동**
- 4개 휠의 회전 방향과 속도 조합으로 **전후·좌우·제자리 회전·대각선 이동** 모두 가능
- 차량의 회전 반경 ≈ 0 (제자리 회전 가능) 으로 **기동성 매우 우수**

### ② 애커만(Ackermann) 섀시 차량

- 일반 자동차와 동일한 조향 방식 — **앞바퀴는 조향, 뒷바퀴는 구동(또는 반대)**
- 차량의 무게는 주로 **뒷바퀴가 지탱**, 앞바퀴는 회전 각도를 조절
- **부드러운 회전(곡선 주행)** 에 유리
- 단점: 제자리 회전 불가, 최소 회전 반경 존재

🟢 **보충** — 애커만 조향은 좌우 앞바퀴의 조향각이 살짝 다릅니다(안쪽 바퀴가 더 큰 각도). 이는 회전 시 네 바퀴 모두 **공통의 순간 회전 중심(ICR)** 을 갖게 하여 타이어 마모를 줄이기 위함입니다.

### ③ 궤도(Track / Tracked) 차량

- 기존 바퀴 대신 **연결된 체인 링크로 구성된 궤도(track)** 사용
- **접지 면적이 넓어** 험지·경사·연약 지반에서 우수한 접지력
- 진동 흡수(서스펜션) 능력이 좋아 울퉁불퉁한 노면에서 유리
- 단점: 평지에서 효율 낮음, 회전 시 마찰 큼, 속도 느림

### 세 휠 유형 비교

| 항목 | 메카넘 | 애커만 | 궤도 |
| :--- | :---: | :---: | :---: |
| 전방향 이동 | ✅ | ❌ | ❌ |
| 제자리 회전 | ✅ | ❌ | ✅ |
| 곡선 주행 부드러움 | △ | ✅ | △ |
| 험지 적응성 | ❌ | △ | ✅ |
| 평지 효율 | ✅ | ✅ | ❌ |
| 미끄럼 손실 | 큼 | 작음 | 중간 |

## 1.2 메카넘 휠의 하드웨어 구조

### 구성 요소

메카넘 휠은 두 가지 주요 부품으로 구성됩니다.

| 부품 | 역할 |
| :--- | :--- |
| **차축 (휠 본체, Hub)** | 전체 휠의 주 지지 구조. 모터 구동축에 직접 결합되며 휠의 회전을 담당 |
| **롤러 (Roller)** | 차축 외주에 부착되는 작은 보조 휠. 자유 회전 가능 |

### 핵심 기하 — 45° 롤러

> [!IMPORTANT]
> 롤러의 회전축은 **휠 본체의 회전축에 대해 45° 기울어져** 부착됩니다. 이 45° 가 메카넘 휠의 핵심 기하학적 특성입니다.

```text
        휠 회전축
          │
          │
    ┌─────┴─────┐
    │     ╱     │      ← 롤러가 45°로 기울어 부착
    │    ╱      │      
    │   ╱       │      이 기울기 덕분에 휠이 회전할 때
    │  ╱  45°   │      추진력이 직진 방향이 아닌
    └───────────┘      45° 비스듬한 방향으로 작용
```

🟢 **보충 — 왜 45° 인가**

- 0° (롤러가 휠과 평행) → 일반 휠과 동일, 측면 이동 불가
- 90° (롤러가 휠과 수직) → 휠이 회전해도 차축 방향으로만 미끄러져 추진 불가
- **45°** → 휠 추진력이 X축 성분과 Y축 성분으로 **반반씩 분해** 되어, 4개 휠의 조합으로 임의 방향 이동 가능

학술 문헌에서는 임의 각도 δ 에 대한 일반 모델을 다루지만, 실용적으로는 거의 모든 메카넘 휠이 **±45°** 로 제작됩니다.

## 1.3 메카넘 휠 섀시의 배치 조합

### 두 가지 휠 타입

> [!NOTE]
> 메카넘 휠은 롤러가 기울어진 방향에 따라 두 종류로 나뉩니다.
> - **A 타입 (좌수계, Left-handed)** : 롤러가 `\` 방향으로 기울어짐
> - **B 타입 (우수계, Right-handed)** : 롤러가 `/` 방향으로 기울어짐
>
> 두 타입은 **거울 대칭** 관계입니다.

### 4륜 섀시 배치 — 표준 패턴

> [!CAUTION]
> 🔴 **원문 정정** — *"AAAA, BBBB, AABB, ABAB, BABA 등 다양한 조합 가능"* 표현은 **모든 조합이 정상 동작한다는 오해를 줄 수 있습니다.** 실제로는 **특정 배치만이 옴니디렉셔널 운동을 지원** 합니다.

위에서 본 4륜 섀시 표준 배치 (포워드 방향이 위쪽):

```text
       정상 동작 패턴 (옴니 가능)            비정상 패턴 (옴니 불가)
    ┌─────────────┐                    ┌─────────────┐
    │  A       B  │  ← 앞              │  A       A  │
    │             │                    │             │
    │  B       A  │  ← 뒤              │  A       A  │
    └─────────────┘                    └─────────────┘
    각 대각선이 같은 타입                  모두 같은 타입 → 일반 휠 4개와 동일

    또는 거울상:
    ┌─────────────┐
    │  B       A  │
    │             │
    │  A       B  │
    └─────────────┘
```

> [!IMPORTANT]
> **올바른 배치의 핵심 규칙**
> - 같은 대각선상의 두 휠은 **같은 타입** (위 그림에서 좌상-우하가 A, 우상-좌하가 B)
> - 좌우 옆쪽으로 인접한 두 휠은 **다른 타입**
>
> 원문의 *"섀시 조합은 ABAB"* 는 **좌측 두 휠이 A, 우측 두 휠이 B** 처럼 좌→우 순서로 읽었을 때의 표기로 해석할 수 있습니다. 다만 위 다이어그램의 대각선 규칙으로 이해하는 것이 더 직관적입니다.

🟢 **보충 — 왜 대각선 규칙인가**

옴니디렉셔널 이동을 위해서는 4개 휠의 추진력 벡터가 X축과 Y축 양쪽으로 적절히 분해되어 합산되어야 합니다. 같은 타입 휠끼리 대각선으로 배치하면 4개 추진력이 **회전 대칭성** 을 가져 임의 방향 이동이 수학적으로 가능해집니다.

## 1.4 전방향 운동의 원리

### 핵심 메커니즘

> [!NOTE]
> 메카넘 휠 섀시의 전방향 이동은 **"4개 휠의 회전 방향과 속도를 조합하여 지면에 작용하는 추진력 벡터의 합을 임의의 방향으로 만드는 것"** 으로 구현됩니다.

각 휠이 회전할 때:
1. 휠 자체는 차축 방향으로 회전
2. 지면과 접촉한 **롤러는 45°로 기울어져 있어** 추진력이 비스듬한 방향으로 작용
3. 4개 휠의 비스듬한 추진력이 **벡터 합산** 되어 차체의 최종 운동 방향 결정

### 기본 운동 패턴

| 목표 운동 | 4개 휠 회전 방향 |
| :--- | :--- |
| **전진** | 4개 휠 모두 전방 회전 |
| **후진** | 4개 휠 모두 후방 회전 |
| **우측 평행 이동** | 좌상 전방, 우상 후방, 좌하 후방, 우하 전방 |
| **좌측 평행 이동** | 위 패턴의 반대 |
| **제자리 시계 회전** | 좌측 두 휠 전방, 우측 두 휠 후방 |
| **제자리 반시계 회전** | 좌측 두 휠 후방, 우측 두 휠 전방 |
| **대각선 이동** | 두 휠만 회전 (대각선상의 두 휠) |

### 🟡 원문 표현 보정

> 원문: *"둘레가 작은 메카넘 휠은 측면으로 미끄러지는 능력이 있음"*

이 표현은 부정확합니다. **휠 둘레의 크기와 측면 이동 능력은 무관** 합니다. 정확한 의미는:

> ✅ **정정** : *"메카넘 휠은 휠 외주의 롤러들이 자유 회전(passive rotation)하므로, 차축 방향으로 능동 회전하는 동시에 롤러 축 방향으로 자유롭게 미끄러질 수 있다. 이 두 자유도의 조합이 측면 이동을 가능하게 한다."*

🟢 **보충 — 추진력 방향 분해**

각 롤러는 휠 회전 시:
- **롤러 축 방향** : 자유 회전이므로 마찰력 없음 (수동적으로 미끄러짐)
- **롤러 축 수직 방향** : 지면과 마찰 → 추진력 발생

롤러 축이 45°이므로, 추진력은 정확히 휠 진행 방향에서 45° 기울어진 방향으로 발생합니다. 이것이 메카넘 휠 특유의 거동을 만드는 원인입니다.

## 1.5 운동학적 모델의 매개변수

메카넘 휠 차체의 운동을 수학적으로 다루기 위해 정의되는 변수들입니다.

### 차체 속도 (몸체 좌표계 기준)

| 기호 | 의미 |
| :--- | :--- |
| **`Vx`** | 차체의 X축 속도 (앞뒤 방향, 전진이 양수) |
| **`Vy`** | 차체의 Y축 속도 (좌우 방향, 좌측 이동이 양수 — ROS 표준) |
| **`Vω`** (또는 `ω_z`) | 차체의 각속도 (반시계 회전이 양수) |

### 차체 기하 매개변수

| 기호 | 의미 |
| :--- | :--- |
| **`L`** | **좌우** 휠 중심 사이의 거리 (차체 폭 방향, **트랙(track)**) |
| **`H`** (또는 `2·l_x`) | **앞뒤** 휠 중심 사이의 거리 (차체 길이 방향, **휠베이스(wheelbase)**) |
| **`r`** | 휠 반지름 |
| **`θω`** | 차체 중심에서 우측 앞 휠 중심을 잇는 선과 X축이 이루는 각도 (정사각 배치 시 45°) |

### 휠 속도 (4개)

각 휠의 회전 각속도(`rad/s`)를 다음과 같이 표기합니다.

| 기호 | 위치 |
| :--- | :--- |
| **`A`** 또는 **`ω_fl`** (front-left) | 좌측 앞 휠 |
| **`B`** 또는 **`ω_fr`** (front-right) | 우측 앞 휠 |
| **`C`** 또는 **`ω_rl`** (rear-left) | 좌측 뒤 휠 |
| **`D`** 또는 **`ω_rr`** (rear-right) | 우측 뒤 휠 |

### 보조 변수 (오른쪽 앞바퀴 운동 분해)

| 기호 | 의미 |
| :--- | :--- |
| **`VBx`** | 오른쪽 앞바퀴 중심의 X 방향 속도 성분 |
| **`VBy`** | 오른쪽 앞바퀴 중심의 Y 방향 속도 성분 |

🟡 **참고** — 원문에서는 `VBx, VBy` 가 *"메카넘 휠의 X/Y축 속도"* 로 표기되어 있지만, 맥락상 **차체 중심이 아닌 특정 휠(우측 앞바퀴) 중심점의 속도** 로 해석됩니다. 변수명 첫 글자 `B` 가 이를 시사합니다 (오른쪽 = right = B).

## 1.6 운동학 공식 유도

### 핵심 가정 (단순화)

수학적 모델 단순화를 위해 다음 두 가지 이상적 가정을 도입합니다.

> [!IMPORTANT]
> **가정 ①** : 모든 바퀴는 지면에서 미끄러지지 않으며(no slip), 마찰력이 충분하다.
>
> **가정 ②** : 4개의 바퀴는 직사각형(또는 정사각형)의 모서리에 배치되어 있으며, 서로 평행하다.

🟢 **보충 — 왜 이런 가정이 필요한가**

- **무미끄럼 가정** : 실제로 메카넘 휠은 옴니 이동 시 일정량의 미끄럼이 필연적이지만 (특히 대각선 이동 시), 이를 무시하면 운동학식이 선형이 되어 분석이 쉬워집니다.
- **사각 배치 가정** : 비대칭 배치에서는 회전 시 회전 중심이 차체 기하 중심에서 벗어나는 보정항이 추가됩니다.

### 운동 분해

차체의 임의 운동은 **세 가지 기본 운동의 선형 합** 으로 분해됩니다.

| 기본 운동 | 차체 속도 |
| :--- | :--- |
| ① X+ 방향 직진 | `(Vx, 0, 0)` |
| ② Y+ 방향 평행 이동 | `(0, Vy, 0)` |
| ③ Z+ 축 회전(반시계) | `(0, 0, Vω)` |

각 기본 운동에 대해 4개 휠의 회전 속도 기여분을 계산한 뒤 합산합니다.

### 휠별 속도 분해 (오른쪽 앞바퀴 예시)

차체가 `(Vx, Vy, Vω)` 로 운동할 때, 우측 앞바퀴의 중심점 속도는:

```text
VBx = Vx − Vω · (L/2)        ← Vω로 인한 회전이 우측 앞바퀴를 +X 또는 -X로 끌어당김
VBy = Vy + Vω · (H/2)        ← 회전이 우측 앞바퀴를 +Y 또는 -Y로 끌어당김
```

(부호 규약은 좌표계와 회전 방향 정의에 따라 책마다 조금씩 다름)

이렇게 구한 휠 중심 속도를 롤러 축에 투영하면 해당 휠의 회전 각속도를 얻을 수 있습니다.

## 1.7 역운동학(Inverse Kinematics) 결과식

> [!NOTE]
> **역운동학(IK)** : 원하는 차체 속도 `(Vx, Vy, Vω)` 로부터 **각 휠에 명령할 회전 각속도 `(A, B, C, D)` 를 계산** 하는 식. 모터 제어에 사용됩니다.

### 표준 형태 (반지름 `r`, 트랙 `L`, 휠베이스 `H` 인 경우)

```text
A (좌측 앞, ω_fl) = (1/r) · ( Vx − Vy − (L+H)/2 · Vω )
B (우측 앞, ω_fr) = (1/r) · ( Vx + Vy + (L+H)/2 · Vω )
C (좌측 뒤, ω_rl) = (1/r) · ( Vx + Vy − (L+H)/2 · Vω )
D (우측 뒤, ω_rr) = (1/r) · ( Vx − Vy + (L+H)/2 · Vω )
```

> [!CAUTION]
> 위 식의 부호 패턴은 *"롤러가 어느 쪽으로 45° 기울어졌는지"* (A 타입/B 타입 배치) 와 *"좌표계가 ROS 표준인지 (X 전방, Y 좌측)"* 에 따라 달라집니다. **자신의 로봇 좌표계와 휠 배치를 정확히 확인한 뒤 부호를 검증** 하세요.

### 변수 의미 재정리

| 기호 | 의미 |
| :--- | :--- |
| `A, B, C, D` | 4개 휠의 **회전 각속도(rad/s)** — 모터에 보내는 명령값 |
| `Vx` | 차체 **X축 직진 속도(m/s)** |
| `Vy` | 차체 **Y축 평행이동 속도(m/s)** |
| `Vω` | 차체 **Z축 회전 각속도(rad/s)** |
| `r` | 휠 반지름(m) |
| `L` | 트랙 (좌우 휠 간격, m) |
| `H` | 휠베이스 (앞뒤 휠 간격, m) |

### 🟢 보충 — 정운동학(Forward Kinematics)

반대로 4개 휠의 회전 속도로부터 차체 속도를 계산하는 식 (오도메트리에 사용):

```text
Vx = (r/4) · ( A + B + C + D )
Vy = (r/4) · ( −A + B + C − D )
Vω = (r / (4·(L+H)/2)) · ( −A + B − C + D )
```

이 두 쌍의 식이 **메카넘 휠 운동학의 핵심** 이며, JetRover_Mecanum 같은 실제 로봇의 구동 노드에 그대로 구현되어 있습니다.

### 검증자 코멘트

원문 메모는 메카넘 휠의 핵심 개념을 거의 정확히 다루고 있습니다. 두 가지 미흡한 부분만 정리하면 완벽한 학습 자료가 됩니다.

1. **휠 배치 조합** — *"AAAA, BBBB, AABB, ABAB, BABA 등 다양한 조합 가능"* 은 마치 모든 조합이 옴니 운동을 지원하는 것처럼 들립니다. 실제로는 **대각선상의 두 휠이 같은 타입(거울상)** 이어야 옴니 이동이 가능합니다. 이는 메카넘 휠의 핵심 설계 규칙이므로 반드시 알아두어야 합니다.

2. **"둘레가 작은 메카넘 휠은 측면으로 미끄러지는 능력이 있음"** — 이 문장은 의미가 통하지 않습니다. 측면 이동의 원인은 **롤러의 자유 회전 + 45° 기울기** 이지 휠 둘레 크기와는 무관합니다. 학습 시 이 부분은 *"롤러가 자유 회전하므로 측면 미끄러짐이 발생하고, 4개 휠의 추진력 벡터를 합산하여 임의 방향 이동을 만든다"* 로 이해하시면 됩니다.

운동학 공식은 변수 정의와 가정이 정확하므로 그대로 코드 구현에 사용 가능합니다. 실제 코드 작성 시에는 부호 규약(좌표계 방향, 회전 양의 방향, 휠 타입 배치)을 자신의 로봇에 맞게 한 번 더 검증하시면 됩니다.
